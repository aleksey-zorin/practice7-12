## Описание проекта

Fullstack-приложение интернет-магазина спортивных товаров с аутентификацией, refresh-токенами и ролевой моделью (RBAC).

---

## Функционал

### Аутентификация (Практика №7-8)
- Регистрация пользователя (email, пароль, имя, фамилия)
- Вход в систему с получением JWT токена
- Защищённые маршруты

### Refresh-токены (Практика №9)
- Access-токен живёт 15 минут
- Refresh-токен живёт 7 дней
- Автоматическое обновление токенов через interceptors в axios

### Фронтенд с хранением токенов (Практика №10)
- Хранение токенов в localStorage
- Axios interceptors для автоматической подстановки токена
- Страницы входа и списка товаров

### RBAC — ролевая модель (Практика №11)

| Роль | Права |
|------|-------|
| **Пользователь** | Просмотр товаров |
| **Продавец** | Просмотр + создание/редактирование товаров |
| **Администратор** | Всё + управление пользователями |

## API Endpoints

### Auth
| Метод | Маршрут | Описание |
|-------|---------|----------|
| POST | `/api/auth/register` | Регистрация |
| POST | `/api/auth/login` | Вход |
| POST | `/api/auth/refresh` | Обновление токенов |
| GET | `/api/auth/me` | Текущий пользователь |

### Products
| Метод | Маршрут | Доступ |
|-------|---------|--------|
| GET | `/api/products` | Все |
| GET | `/api/products/:id` | Все |
| POST | `/api/products` | Seller, Admin |
| PUT | `/api/products/:id` | Seller, Admin |
| DELETE | `/api/products/:id` | Admin |

### Users (только Admin)
| Метод | Маршрут | Описание |
|-------|---------|----------|
| GET | `/api/users` | Список пользователей |
| GET | `/api/users/:id` | Пользователь по ID |
| PUT | `/api/users/:id` | Обновление пользователя |
| DELETE | `/api/users/:id` | Блокировка |
